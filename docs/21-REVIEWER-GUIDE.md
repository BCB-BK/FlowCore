# Reviewer- und Genehmiger-Leitfaden

## Ihre Rolle

### Reviewer
Als Reviewer prüfen Sie eingereichte Revisionen auf:
- Fachliche Korrektheit
- Vollständigkeit
- Verständlichkeit
- Einhaltung von Standards

### Approver (Genehmiger)
Als Genehmiger erteilen Sie die finale Freigabe für die Veröffentlichung.

## Workflow

### Schritt 1: Benachrichtigung
Sie werden benachrichtigt, wenn eine Revision zur Prüfung eingereicht wird.
Ihre ausstehenden Reviews finden Sie unter **"Meine Aufgaben"**.

### Schritt 2: Prüfung
1. Öffnen Sie die Seite mit der eingereichten Revision
2. Wechseln Sie zur **Revisionshistorie**
3. Prüfen Sie die Änderungen im **Diff-View**
4. Achten Sie auf:
   - Fachliche Richtigkeit der Inhalte
   - Vollständigkeit aller Pflichtfelder
   - Korrekte Metadaten (Owner, Tags, Gültigkeitsdatum)
   - Verständliche und fehlerfreie Texte
   - Korrekte Verlinkungen

### Schritt 3: Entscheidung

#### Genehmigen
Wenn die Revision den Anforderungen entspricht:
1. Klicken Sie auf **"Genehmigen"**
2. Optional: Fügen Sie einen Kommentar hinzu
3. Die Revision ist nun bereit zur Veröffentlichung

#### Änderungen anfordern
Wenn Überarbeitungen nötig sind:
1. Klicken Sie auf **"Änderungen anfordern"**
2. Beschreiben Sie konkret, was geändert werden muss
3. Die Revision geht zurück an den Editor

#### Ablehnen
Bei grundlegenden Problemen:
1. Klicken Sie auf **"Ablehnen"**
2. Begründen Sie die Ablehnung
3. Der Editor muss eine neue Revision erstellen

## Prüfkriterien

### Inhaltliche Prüfung
- [ ] Fachlich korrekt und aktuell
- [ ] Keine widersprüchlichen Aussagen zu bestehenden Inhalten
- [ ] Prozessschritte vollständig und nachvollziehbar
- [ ] Zuständigkeiten klar zugeordnet

### Formale Prüfung
- [ ] Richtiger Seitentyp gewählt
- [ ] Alle Pflichtfelder ausgefüllt
- [ ] Tags zugewiesen
- [ ] Verantwortlicher (Owner) eingetragen
- [ ] Nächstes Prüfdatum gesetzt

### Qualitätsprüfung
- [ ] Klare, verständliche Sprache
- [ ] Konsistente Formatierung
- [ ] Funktionelle Verlinkungen
- [ ] Bilder/Medien relevant und beschriftet

## Qualitäts-Dashboard

### Zugang
Reviewer mit der Berechtigung `view_quality_dashboard` können das **Qualitäts-Dashboard** einsehen:
- **Seitenqualität**: Übersicht der Qualitätsbewertung aller Seiten
- **Wartungshinweise**: Seiten mit Handlungsbedarf
- **Duplikate**: Potenzielle Dopplungen
- **Suchanalyse**: Meistgesuchte Begriffe und Null-Ergebnis-Suchen

### Qualitätskriterien (5 Dimensionen × 20 Punkte)
1. **Verantwortlicher**: Ist ein Owner zugewiesen?
2. **Aktuelle Revision**: Gibt es eine aktuelle Revision?
3. **Veröffentlicht**: Ist eine Version veröffentlicht?
4. **Tags**: Sind Tags zugewiesen?
5. **Review-Aktualität**: Ist das Prüfdatum nicht überschritten?

## Tipps

1. **Zeitnah prüfen**: Offene Reviews blockieren die Veröffentlichung
2. **Konstruktives Feedback**: Nennen Sie konkrete Verbesserungsvorschläge
3. **Konsistenz wahren**: Achten Sie auf einheitliche Standards
4. **Verknüpfungen prüfen**: Stellen Sie sicher, dass Links funktionieren
5. **Aktualität**: Achten Sie auf veraltete Referenzen und Daten
